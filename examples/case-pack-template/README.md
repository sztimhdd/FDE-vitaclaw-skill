# Case Pack Template

Use this structure when a live project should stress-test the skill and leave behind reusable assets.

```text
case-packs/
└── your-domain-case/
    ├── raw-context.md
    ├── operator-artifacts/
    │   ├── 01-mission-brief.md
    │   ├── 02-operational-reality-map.md
    │   ├── 03-case-replay-pack.md
    │   ├── 04-system-problem-frame.md
    │   ├── 05-state-action-evidence-model.md
    │   ├── 06-ai-intervention-design.md
    │   ├── 07-minimum-viable-loop.md
    │   ├── 08-poc-acceptance-contract.md
    │   ├── 09-eval-pack.md
    │   ├── 10-governance-and-risk-overlay.md
    │   ├── 11-day-2-operations-plan.md
    │   └── 12-expansion-roadmap.md
    ├── reusable-patterns.md
    ├── eval-cases.md
    ├── product-feedback.md
    └── asset-distillation-log.md
```

## Rule

Keep project-private evidence in the case pack. Promote only cleaned and reusable assets back into the core skill.

## Minimum Distillation Outputs

- one reusable problem pattern
- one eval or failure-case pack
- one product, tooling, or SOP implication
- one explicit promotion decision with owner
