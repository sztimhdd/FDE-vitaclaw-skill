# PRD Writer

## Purpose

Turn FDE artifacts into an implementation-ready product requirements document.

## Use When

- mission and loop are qualified
- operational reality is understood
- core objects and source rules are defined
- an engineering team or coding agent needs build-ready context

## Required Sections

1. Executive Summary
2. User Persona
3. Problem And Current Workflow
4. Source Materials Or Systems
5. Core Object Model
6. Field Or Decision Rules
7. Workflow
8. Operator Interaction
9. Data Model Direction
10. Error Handling
11. Eval Criteria
12. Out Of Scope
13. Future Work

## Quality Bar

The document is acceptable only when another builder can identify:

- what to build
- what not to build
- which rules matter
- how it will be tested
- which unknowns remain
