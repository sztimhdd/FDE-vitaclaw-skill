# POC Acceptance Contract

## Use

Use to lock how the pilot will be judged before it is built or demoed.

## Required Fields

- POC objective
- Success criteria
- Failure criteria
- Demo path
- Required evidence
- Known exclusions
- Decision after review

## Optional Fields

- Confidence target
- Reviewer list
- Escalation triggers

## Typical Anti-Pattern

The team builds first and lets stakeholders redefine success after the demo.

## Completion Standard

Success and failure are judgeable by an external reviewer without rewriting the criteria.
