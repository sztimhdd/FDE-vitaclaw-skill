# Day-2 Operations Plan

## Use

Use when the team must define how the loop is operated after the demo or pilot launch.

## Required Fields

- Operating owner
- Exception owner
- Support path
- Monitoring signals
- Drift or degradation signal
- Manual fallback mode
- Review cadence
- Issue triage rule
- Change approval rule
- Maintenance burden
- Operator adoption signal
- Feedback capture method
- Asset promotion candidate

## Optional Fields

- Staffing assumption
- Escalation SLA
- Known weak points

## Typical Anti-Pattern

The pilot works in a demo but no one owns exceptions, degradation, review, support, or operator adoption after launch.

## Completion Standard

The loop can run for 30 days without the original builder being present in every exception.
