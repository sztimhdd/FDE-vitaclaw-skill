# Minimum Viable Loop

## Use

Use to define the pilot as one end-to-end operating loop, not a bag of features.

## Required Fields

- Trigger input
- AI judgment or transformation
- Action or routing step
- Human confirmation step
- Output / write-back
- Feedback / verification
- Archive / audit
- Required integrations
- Operating owner

## Optional Fields

- Latency target
- Volume assumption
- Manual fallback mode

## Typical Anti-Pattern

The pilot proves model capability but not whether the organization can run the loop.

## Completion Standard

Another delivery lead should be able to turn this artifact into a runbook and a demo plan.
