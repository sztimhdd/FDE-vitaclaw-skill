# Operating Principles

## 1. Loop before platform

Always identify one operator, one trigger, one business object, and one output before proposing a broader system.

## 2. Evidence before confidence

Separate what the customer provided from what the agent inferred.

## 3. Source of truth before validation

When documents, systems, or people disagree, define which source controls which field or decision.

## 4. Human boundary before automation

Identify what remains advisory, what requires approval, and what can be automated later.

## 5. Acceptance before implementation

Do not hand work to a coding agent until acceptance criteria, tests, and evidence requirements exist.

## 6. Benchmark outside doctrine

Do not contaminate generic skill behavior with benchmark-specific facts.

## 7. Handoff as contract

Downstream coding agents need explicit context, constraints, tests, and rollback notes.
