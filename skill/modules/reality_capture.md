# Reality Capture

## Purpose

Reconstruct how the work actually happens today.

## Use When

- the user describes an operational workflow
- the real process differs from the official process
- documents, screenshots, spreadsheets, or conversations define the work
- exceptions and handoffs matter

## Required Capture

- actors and owners
- clean case
- ugly case
- disputed case
- systems and files used
- shadow workflow
- evidence available at decision time
- failure points

## Output

Produce an Operational Reality Map.

## Stop Condition

Stop when another operator can replay the workflow without interviewing the original stakeholder.
